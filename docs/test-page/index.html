#通过 MWEB 结合 github page 进行快速发布

##1. 图片快速保存 jekyll 要求图片保存在根目录的 assets 中，但是全部保存在里面有些混乱，我们一般希望对不同的文章，设置不同的文件夹进行管理

  1. 首先打开mweb 的三栏模式，将 jekyll 的根目录拖拽进去 比如我这里，github page 是通过 master 分支下面的 docs 文件夹生成的，那么就将 docs 文件夹全部拖入

  2. 双击 docs 文件夹，进行文件夹设置
    这里我们需要选择绝对位置，然后在保存拖入的图片文件夹名称中，填写需要保存的文件夹名，这里我使用了

     assets/post_image/test-page

    表示我的图片在拖入或者粘贴到编辑器的时候，自动保存一个备份文件到如下目录，assets是 jekyll 要求的，下面的次级目录根据自己的需要生成，我将全部的文章图片放在post_image中，然后 test-page是我这篇文章的permalink

     ---
 layout: post
 title: Test Page
 date:   2015-06-04 13:50:39
 categories: others
 permalink : /test-page/
 ---

    如果你不设置 Permalink 的时候，jekyll 生成的这篇 markdown 一般是以日期和标题缩写构成的，这样就不方便我们进行资源的定位。所以这种方法要求大家一定要给每一篇文章设置permalink，并且在编辑对于文章的时候，要修改一下保存文件的目录

    当你设置了permalink之后，在生成的_site文件夹中，会有一个专门的文件夹来保存渲染后的 html 文件，不过这些都是自动生成的，也不需要你去编辑，所以不会造成什么麻烦，所有的文档仍然在 _posts 中保存
